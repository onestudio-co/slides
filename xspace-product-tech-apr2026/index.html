07 / Open Items — Security
Security — Must Close Before Investor Due Diligence

Security Findings

🔴
Firebase private key committed to public repo
Service account key exposed in git history since January 19 — 76 days. Any actor with the commit history has full Firebase project access.
Fix: Rotate key immediately. Clean git history (BFG / filter-branch). Confirm no unauthorized access in Firebase audit log.
🔴
Tap Payments API keys hardcoded in config defaults
Production payment keys visible in source. If the production key is active, any merchant-level fraud is possible.
Fix: Move all keys to environment variables. Verify production key status with Tap directly.
🔴
OTP test mode may be active in production
OTP_TEST_MODE=true with hardcoded code 1234 in .env.example. If this leaked into production .env, any account is bypassable.
Fix: Confirm production .env. Enforce OTP_TEST_MODE=false in deployment checklist.
🔴
Mobile API key hardcoded in source
API key compiled into the APK binary. Extractable by any user with the APK.
Fix: Implement certificate pinning + backend API key rotation strategy.
🔴
Payment webhook — no HMAC signature verification
Tap Payments webhook endpoint accepts payloads without verifying origin. Spoofed payment confirmations are possible.
Fix: Implement Tap's HMAC header verification. 2–4 hour fix. High priority.
