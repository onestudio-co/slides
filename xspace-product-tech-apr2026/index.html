04 / What We Must Close Now
3 Gates — Must Close Before Phase 2

These block investor due diligence and Phase 2 launch

1
Security — Exposed credentials in the codebase
A Firebase service account key and payment API keys are visible in the code repository — have been for 76 days. Any investor or partner who views the repo before this is fixed can see them. Standard due diligence failure point.
Action: Rotate all keys. Clean repository history. Verify no unauthorized access. — 1 week.
1 week
2
IP Ownership — No signed assignments from developers
The 4 developers who built this product have not signed IP assignment documents. Without these, xSpace does not legally own the code — even though it paid for it. This is a blocker for any fundraising or acquisition conversation.
Action: IP assignment letter signed by all 5 developers. Firebase, Tap Payments, App Store transferred to xSpace entity. — Runs in parallel, 1–2 weeks.
1–2 wks
3
Payment Gateway — Tap confirmation & Moyasar decision
Tap Payments is integrated. We need confirmation that the production merchant account is active and live. Moyasar — the preferred Saudi-native gateway — is not yet integrated. Without a confirmed live gateway, the product cannot process real bookings.
Action: Confirm Tap production status this week. Decide Moyasar integration timeline (2–3 weeks to build).
This week
All 3 gates run in parallel — estimated total time to clear all:
2–3 weeks
